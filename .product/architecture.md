## Architecture Pattern

Next.js 16 App Router with **route groups** for role-based layouts. Each role (`admin`, `teacher`, `student`) has its own layout with dedicated sidebar navigation. Server Components by default, `'use client'` where interactivity is needed. No separate backend — API routes colocated in `app/api/`.

## Import/Dependency Flow

```
UI Pages → Components → lib/ (auth, payments, supabase) → Supabase Cloud (PostgreSQL)
API Routes → lib/ (auth, payments, quiz, placement) → Supabase Cloud
Middleware → Supabase Auth → Role-based route protection
```
